# Hipparcos + Gaia DR3 星表文件结构详解

## 概述

本目录存储了 Stellarium 采用的 Hipparcos + Gaia DR3 组合星表的所有数据文件，采用多文件模块化存储结构，将不同类型的天文数据分离存储，兼顾加载性能、扩展性和维护便捷性。

## 各类文件详细解析

### 1. 配置文件：defaultStarsConfig.json
#### 核心作用
星表系统的入口配置文件，定义所有星表文件的元数据、加载规则和版本信息，是Stellarium加载星表数据的核心索引。

#### 数据结构
```json
{
  "version": 26,                     // 星表配置版本号
  "hipSpectralFile": "stars_hip_sp_0v0_6.cat",  // 光谱数据文件路径
  "objecttypesFile": "object_types_v0_1.cat",  // 天体类型文件路径
  "catalogs": [                     // 各层级主星表配置
    {
      "id": "stars0",              // 星表唯一标识
      "fileName": "stars_0_0v0_20.cat", // 文件名
      "count": 0.005,               // 恒星数量（百万颗）
      "magRange": [-2.00, 6.00],   // 星等覆盖范围
      "sizeMb": 0.2,               // 文件大小（MB）
      "checksum": "...",           // MD5校验值
      "checked": true              // 是否已安装
    }
  ]
}
```

#### 关键参数说明
- **version**: 配置文件版本号，用于星表更新时的版本校验
- **magRange**: 每个主星表的星等区间，确保星等范围连续无重叠
- **checksum**: 文件校验码，用于下载完整性校验
- **checked**: 标记用户是否已安装该层级星表，支持按需下载扩展

### 2. .cat 格式文件详解
.cat 为 Stellarium 自定义的二进制紧凑存储格式，根据存储内容不同分为多个子类：

#### 2.1 层级主星表 (stars_X_YvZ_N.cat)
这是之前已详细解读的核心星表文件，采用分层存储策略：
- **分层规则**: 按星等从亮到暗分为0~8共9个层级
- **数据结构**: 分为 Type0 (48字节/记录，level 0-3) 和 Type1 (32字节/记录，level 4-8) 两种格式
- **核心字段**: 位置坐标、自行、视差、星等、色指数、光谱类型索引等
- **命名规则**: `stars_{level}_{datatype}v{majver}_{minver}.cat`

#### 2.2 光谱类型文件 (stars_hip_sp_0v0_6.cat)
存储 Hipparcos 恒星的详细光谱类型数据，与主星表通过 source_id 关联：
- 数据量: ~12万条 Hipparcos 恒星光谱信息
- 核心字段: Hipparcos ID、光谱类型编码、有效温度、金属丰度等
- 作用: 为主星表的光谱类型字段提供更精确的补充数据

#### 2.3 天体类型分类文件 (object_types_v0_1.cat)
天体类型字典文件，定义了约200种天体类型编码：
```
# 部分类型示例
**  | 双星系统
SB* | 分光双星
WR* | 沃尔夫-拉叶星
RG* | 红巨星
WD* | 白矮星
V*  | 变星
Psr | 脉冲星
BH  | 黑洞候选体
OpC | 疏散星团
GlC | 球状星团
```
- 作用: 将主星表中 `otype_idx` 字段的映射字典
- 优势: 主星表仅存储1字节索引，节省存储空间

#### 2.4 变星总表 (gcvs.cat - General Catalog of Variable Stars)
存储约5万颗变星的完整信息：
- 核心字段: 变星ID、变星类型、光变周期、极大星等、极小星等、光谱变化范围
- 数据格式: 二进制紧凑存储，与主星表通过 source_id 关联
- 作用: 实现恒星亮度变化的动态渲染

#### 2.5 交叉索引表 (cross-id.cat)
多星表ID映射文件：
- 核心映射: Hipparcos ID ↔ Gaia Source ID ↔ Tycho ID ↔ 其他星表ID
- 作用: 实现不同来源星表的数据关联和融合
- 优势: 主星表仅需存储一个主ID，其他ID通过交叉索引关联查询

### 3. .fab 命名文件详解
.fab 是 Stellarium 自定义的文本格式命名文件，存储恒星名称数据：

#### 3.1 标准命名文件 (name.fab)
存储恒星的官方标准命名：
```text
# 格式: 星表ID | 命名
    677|α_And      # 仙女座α（壁宿二）
   5447|β_And      # 仙女座β（奎宿九）
  3092|δ_And      # 仙女座δ
32349|Sirius     # 天狼星
```
- 包含内容: 拜耳命名（希腊字母+星座）、弗兰斯蒂德编号（数字+星座）、变星命名
- 数据量: 约3万条正式命名恒星

#### 3.2 扩展命名文件 (extra_name.fab)
存储恒星的扩展名称：
- 包含内容: 恒星专有名称（如 Sirius、Betelgeuse）、俗称、历史名称、多语言名称
- 作用: 提供更丰富的恒星名称显示选项

---

### 4. .dat 轨道参数文件详解
.dat 为二进制格式的特殊天体数据文件：

#### 4.1 双星轨道参数文件 (binary_orbitparam.dat)
存储已精确测量的双星系统轨道参数：
```text
# 字段说明:
primary_hip  secondary_hip  period  ecc  inc  omega  Omega  T0  a  distance  epoch  ra  dec  rv  pmra  pmdec  mass1  mass2
```
- 核心参数:
  - `period`: 轨道周期（天）
  - `ecc`: 轨道偏心率
  - `inc`: 轨道倾角（弧度）
  - `a`: 轨道半长轴（毫角秒）
  - `mass1/mass2`: 主/伴星质量（太阳质量单位）
- 作用: 实现双星系统的轨道运动模拟和渲染

#### 4.2 华盛顿双星目录文件 (wds_hip_part.dat)
存储华盛顿双星目录（WDS）中与 Hipparcos 星表重合的双星数据：
- 包含约10万对双星系统的基本信息
- 与 binary_orbitparam.dat 形成互补，提供更全面的双星数据

## 多文件存储结构设计分析

### 设计考量
#### 1. 按需加载优化
- **场景适配**: 不同配置文件中 `checked` 字段支持用户按需下载不同层级的星表，默认仅安装最亮的4个层级（约60万颗星），完整星表2.1亿颗可按需扩展
- **内存优化**: 运行时仅加载当前视场范围内需要的星表层级，避免加载全量数据占用内存
- **启动速度**: 分层加载显著减少程序启动时间

#### 2. 模块化维护优势
| 优势 | 说明 |
|------|------|
| **数据更新便捷** | 某类数据更新时无需重新发布整个星表，例如更新变星数据时仅需更新 `gcvs.cat` |
| **扩展性强** | 新增天体类型时仅需更新 `object_types_v0_1.cat`，不影响其他文件 |
| **调试便捷** | 问题定位方便，不同类型数据问题可单独排查对应文件 |
| **多源数据融合** | 不同来源数据（Hipparcos/Gaia/GCVS/WDS）独立存储，保持数据来源可追溯 |

#### 3. 性能优化
- **IO效率**: 二进制文件比文本文件读写速度快5~10倍
- **存储效率**: 分层存储+量化编码+zlib压缩，总存储效率提升约80%
- **缓存友好**: 小文件更适合操作系统文件系统缓存，频繁访问的元数据文件（如命名文件、分类文件）可常驻缓存

#### 4. 兼容性设计
- **版本化命名**: 所有文件均包含版本号（如 `v0_20`），支持新旧版本星表共存
- **向后兼容**: 新增字段时可新增独立文件，不影响旧版本软件读取
- **增量更新**: 支持仅下载更新变化的文件，无需全量下载

### 技术实现逻辑
```
星表加载流程:
1. 读取 defaultStarsConfig.json 配置文件
2. 根据配置加载已安装的主星表文件头信息
3. 根据用户视场范围加载对应层级的恒星数据
4. 按需加载关联数据（名称、光谱、轨道参数等）
5. 数据融合后送入渲染管线

数据关联逻辑:
主星表记录(source_id) → cross-id.cat → 其他星表关联查询
主星表记录(otype_idx) → object_types_v0_1.cat → 天体类型名称
主星表记录(source_id) → name.fab/extra_name.fab → 恒星名称
主星表记录(source_id) → binary_orbitparam.dat → 双星轨道参数
```

## 与 stars_*.cat 主星表对比分析

| 对比维度 | stars_*.cat 主星表 | 其他辅助星表文件 |
|----------|-------------------|------------------|
| **数据量级 | 核心主数据 | 辅助补充数据 |
| **存储格式** | 分层二进制存储 | 多种格式混合（二进制/文本） |
| **访问频率** | 极高（每一帧渲染都需要访问） | 较低（仅查询时访问） |
| **记录数量** | 大（最多10万~1.2亿条） | 小（几千~几十万条） |
| **更新频率** | 低（星表基础数据稳定） | 中高（补充数据更新频繁） |
| **内存占用** | 大 | 小 |
| **加载时机** | 程序启动/视场变化时加载 | 按需懒加载 |

## 总结

Hipparcos + Gaia DR3 星表的多文件存储结构是 Stellarium 团队在性能、扩展性、维护性三者之间平衡的优秀设计，既满足了天文软件对海量星表数据的高效访问需求，也兼顾了普通用户的设备性能和使用场景。
