# EPH 星表二进制文件格式说明

## 概述
EPH (EPHEmeris) 是 Stellarium Web Engine 使用的二进制星表文件格式，用于高效存储和传输恒星、深空天体等天文数据。该格式设计紧凑、支持压缩，便于快速读取和渲染。

## 文件整体结构
| 偏移 | 长度 | 类型 | 说明 |
|------|------|------|------|
| 0 | 4字节 | ASCII | 文件标识，固定为 "EPHE" |
| 4 | 4字节 | 小端无符号整数 | 文件版本，当前为 2 |
| 8 | 可变 | - | Chunk 列表，文件由一系列 Chunk 组成 |

## Chunk 结构
每个 Chunk 遵循以下格式：
| 偏移 | 长度 | 类型 | 说明 |
|------|------|------|------|
| 0 | 4字节 | ASCII | Chunk 类型，不足4字节用空格补全 |
| 4 | 4字节 | 小端无符号整数 | Chunk 数据长度 (len) |
| 8 | len字节 | - | Chunk 实际数据 |
| 8 + len | 4字节 | 小端无符号整数 | CRC32 校验和（本实现暂时用0填充） |

### 常见 Chunk 类型
1. **JSON** - 元数据存储
   - 内容：UTF-8 编码的 JSON 字符串，存储文件生成信息、版本等元数据
   - 示例：`{"generator":"python", "version":"1.0"}`

2. **STAR** - 恒星数据块
   - 包含恒星Tile头和表格数据
   - 结构详见下文

3. **DSO** - 深空天体数据块
   - 格式与STAR类似，存储深空天体（星系、星云等）数据

4. **COMP** - 压缩数据块
   - 存储zlib压缩的其他Chunk数据
   - 结构：4字节解压后大小 + 4字节压缩后大小 + 压缩数据

5. **TABLE** - 通用表格数据块
   - 存储结构化表格数据，可被压缩或直接包含在STAR/DSO块中

## STAR Chunk 详细结构
STAR Chunk 是最常用的块类型，存储具体的恒星数据：
| 偏移 | 长度 | 类型 | 说明 |
|------|------|------|------|
| 0 | 4字节 | 小端无符号整数 | Tile版本，当前为 3 |
| 4 | 8字节 | 小端无符号整数 | nuniq，HEALPix 层级像素编码 |
| 12 | 可变 | - | TABLE 数据块 |

### TABLE 数据块结构
TABLE 块包含结构化的表格数据：
| 偏移 | 长度 | 类型 | 说明 |
|------|------|------|------|
| 0 | 4字节 | 小端无符号整数 | 标志位 |
| | | | - 第0位：1表示数据经过shuffle（按列重排），0表示原始顺序 |
| 4 | 4字节 | 小端无符号整数 | 每行数据大小（字节） |
| 8 | 4字节 | 小端无符号整数 | 列数 |
| 12 | 4字节 | 小端无符号整数 | 行数 |
| 16 | 列数 × 20字节 | - | 列描述信息，每列20字节 |
| 16 + 列数×20 | 可变 | - | 表格数据（原始数据或压缩块） |

### 列描述信息结构
每列的描述信息占用20字节：
| 偏移 | 长度 | 类型 | 说明 |
|------|------|------|------|
| 0 | 4字节 | ASCII | 列名，不足4字节用NULL补全 |
| 4 | 4字节 | - | 类型字段，首字节为类型标识： |
| | | | - 'f'：float 单精度浮点数 |
| | | | - 'i'：int 无符号整数 |
| | | | - 'Q'：uint64 64位无符号整数 |
| | | | - 's'：string 字符串 |
| 8 | 4字节 | 小端无符号整数 | 单位标识 |
| 12 | 4字节 | 小端无符号整数 | 该列数据在每行中的起始偏移 |
| 16 | 4字节 | 小端无符号整数 | 该列数据占用的字节数 |

### 压缩块结构
如果TABLE数据是压缩的（通常如此），数据部分结构为：
| 偏移 | 长度 | 类型 | 说明 |
|------|------|------|------|
| 0 | 4字节 | 小端无符号整数 | 解压后数据大小 |
| 4 | 4字节 | 小端无符号整数 | 压缩数据大小 |
| 8 | 压缩大小字节 | - | zlib压缩的数据 |

## 数据优化技术
1. **字节重排 (Shuffle)**
   - 为提高压缩率，数据会按列重排
   - 原始按行存储：[行1列1, 行1列2, ..., 行1列N, 行2列1, ...]
   - 重排后按列存储：[行1列1, 行2列1, ..., 行N列1, 行1列2, ...]
   - 读取时需要反向shuffle恢复原始顺序

2. **zlib压缩**
   - 重排后的数据使用zlib压缩，进一步减小文件体积
   - 压缩级别通常使用最高级别(level=9)

## 恒星数据列定义
标准恒星数据包含10列，每行大小292字节：
| 列名 | 类型 | 单位 | 起始偏移 | 大小 | 说明 |
|------|------|------|----------|------|------|
| hip | i | 0 | 0 | 4 | Hipparcos 星表编号 |
| hd | i | 0 | 4 | 4 | Henry Draper 星表编号 |
| vmag | f | 196608 | 8 | 4 | 视星等 |
| ra | f | 65536 | 12 | 4 | 赤经（弧度） |
| de | f | 65536 | 16 | 4 | 赤纬（弧度） |
| plx | f | 65543 | 20 | 4 | 视差 |
| pra | f | 393216 | 24 | 4 | 赤经自行 |
| pde | f | 393216 | 28 | 4 | 赤纬自行 |
| bv | f | 0 | 32 | 4 | B-V 色指数 |
| ids | s | 0 | 36 | 256 | 天体名称/标识字符串 |

## 使用工具
- `read_eph.py`：EPH文件解析和查看工具
  ```bash
  python read_eph.py <filename.eph>
  ```

- `write_eph.py`：EPH文件生成工具
  ```bash
  python write_eph.py <input.txt> <output.eph>
  ```

- `generate_test_data.py`：测试数据生成工具
  ```bash
  python generate_test_data.py
  ```
